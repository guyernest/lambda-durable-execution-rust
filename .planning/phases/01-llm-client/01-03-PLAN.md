<objective>
Implement SecretManager for API key retrieval and UnifiedLLMService that wires together secrets, transformers, and HTTP calls into the complete LLM invocation pipeline.

Purpose: This is the top-level API that the agent loop (Phase 3) will call. `UnifiedLLMService::process(invocation)` is the single entry point that handles the entire flow: authenticate, transform, call provider, parse response. Per D-05, service initialization happens outside durable steps so secrets are never checkpointed.

Output: Complete, compilable LLM client module with service, secrets, and full integration. Ready for Phase 3 to wrap in `ctx.step()`.
</objective>

   - Implement `UnifiedLLMService::new()`:
     ```rust
     pub async fn new() -> Result<Self, LlmError> {
         info!("Initializing UnifiedLLMService");
         let secret_manager = Arc::new(SecretManager::new().await?);
         let http_client = Client::builder()
             .timeout(Duration::from_secs(120))  // LLM calls can be slow
             .connect_timeout(Duration::from_secs(10))
             .pool_max_idle_per_host(10)
             .build()
             .map_err(LlmError::HttpError)?;
         let transformer_registry = Arc::new(TransformerRegistry::new());
         Ok(Self { secret_manager, http_client, transformer_registry })
     }
     ```
     Note: 120s timeout (not 60s from original) because agent LLM calls with tool use can be slow.
   - Implement `process(&self, invocation: LLMInvocation) -> Result<LLMResponse, LlmError>`:
     Follow the exact same 7-step pipeline as the original:
     1. Get API key: `self.secret_manager.get_api_key(&invocation.provider_config.secret_path, &invocation.provider_config.secret_key_name).await?`
     2. Get request transformer: `self.transformer_registry.get(&invocation.provider_config.request_transformer)?`
     3. Transform request: `request_transformer.transform_request(&invocation)?`
     4. Call provider: `self.call_provider(&invocation.provider_config, transformed_request, &api_key).await?`
     5. Get response transformer: `self.transformer_registry.get(&invocation.provider_config.response_transformer)?`
     6. Transform response: `response_transformer.transform_response(response)?`
     7. Build LLMResponse with ResponseMetadata (model_id, provider_id, latency_ms, tokens_used, stop_reason)
   - Implement `call_provider(&self, config: &ProviderConfig, request: TransformedRequest, api_key: &str) -> Result<Value, LlmError>`:

     - Set auth header: `config.auth_header_name` with value = `"{prefix}{key}"` if auth_header_prefix is Some, else just `"{key}"`
     - Add custom_headers from config
     - Add request.headers from transformer
     - Send request, check status
     - On non-success status: return `LlmError::ProviderApiError { provider, status, message: error_body }`
     - On success: parse JSON response body
   - Add a `new_with_components(secret_manager: Arc<SecretManager>, http_client: Client, transformer_registry: Arc<TransformerRegistry>) -> Self` constructor for testing
