<objective>
Create the configuration loading module that reads agent config from DynamoDB AgentRegistry table and maps it to internal types.

Purpose: The agent needs its configuration (system prompt, LLM model, parameters, MCP server URLs) loaded from DynamoDB at startup. This config is loaded inside a durable ctx.step() so it is cached on replay. The config types must be Serialize + Deserialize for checkpoint compatibility.

Output: config/ module with types.rs (AgentConfig, AgentParameters), loader.rs (DynamoDB reader + provider mapping), error.rs (ConfigError), and mod.rs (re-exports). All unit-tested with mock DynamoDB attribute maps.
</objective>

**4. Create `examples/src/bin/mcp_agent/config/loader.rs`:**

Implement three public functions:

a) `load_agent_config(table_name, agent_name, version) -> Result<AgentConfig, ConfigError>`: Creates aws DynamoDB client, calls get_item with agent_name (PK) and version (SK), delegates to parse_agent_config. Per CONF-01.

b) `parse_agent_config(item: &HashMap<String, AttributeValue>) -> Result<AgentConfig, ConfigError>`: Extracts fields from DynamoDB item. Per CONF-02 and CONF-03:
  - `agent_name`: read as S (String)
  - `version`: read as S (String)
  - `system_prompt`: read as S (String)
  - `llm_provider`: read as S (String) -- input to map_provider_config
  - `llm_model`: read as S (String) -- input to map_provider_config
  - `parameters`: read as S (String) containing JSON, parse with serde_json::from_str::<AgentParameters>. Fall back to AgentParameters::default() if field is missing.
  - `mcp_servers`: read as S (String) containing JSON array, parse with serde_json::from_str::<Vec<String>>. Default to empty vec if missing. Per D-01.

Use helper functions:
```rust
fn get_string(item: &HashMap<String, AttributeValue>, key: &str) -> Result<String, ConfigError>
fn get_json_string_as<T: DeserializeOwned>(item: &HashMap<String, AttributeValue>, key: &str) -> Result<T, ConfigError>
fn get_optional_json_string_as<T: DeserializeOwned>(item: &HashMap<String, AttributeValue>, key: &str, default: T) -> T
```

c) `map_provider_config(llm_provider, llm_model) -> Result<ProviderConfig, ConfigError>`: Per D-02, hardcoded mapping:
  - "claude" | "anthropic" => ProviderConfig with provider_id "anthropic", endpoint "https://api.anthropic.com/v1/messages", auth_header_name "x-api-key", auth_header_prefix None, secret_path "prod/anthropic/api-key", secret_key_name "api_key", request_transformer "anthropic_v1", response_transformer "anthropic_v1", timeout 120, custom_headers with "anthropic-version" => "2023-06-01"
  - "openai" => ProviderConfig with provider_id "openai", endpoint "https://api.openai.com/v1/chat/completions", auth_header_name "Authorization", auth_header_prefix Some("Bearer "), secret_path "prod/openai/api-key", secret_key_name "api_key", request_transformer "openai_v1", response_transformer "openai_v1", timeout 120, custom_headers None
  - _ => ConfigError::UnsupportedProvider

Read table name from env var `AGENT_REGISTRY_TABLE` with fallback to `"AgentRegistry"`.
