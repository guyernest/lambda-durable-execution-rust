<objective>
Create the MCP client integration module that connects to MCP servers, discovers tools, translates schemas, and builds a routing map.

Purpose: The agent needs to connect to 1-3 MCP servers at startup, discover their available tools, prefix tool names with host identifiers to avoid collisions, translate MCP tool schemas to the Claude API format (UnifiedTool), and maintain a routing map to dispatch tool calls back to the correct server. Connections are ephemeral (per D-07) but the discovered tool schemas and routing map are serializable for checkpointing.

Output: mcp/ module with client.rs (connection, discovery, tool translation, routing), types.rs (ToolsWithRouting), error.rs (McpError), and mod.rs (re-exports). Pure function tests for extraction/translation/routing; connection tests deferred to Phase 5 integration.
</objective>

**4. Create `examples/src/bin/mcp_agent/mcp/client.rs`:**

Implement the following public functions:

a) `discover_all_tools(server_urls: &[String]) -> Result<ToolsWithRouting, McpError>`:
  - Per MCP-06: If `server_urls` is empty, return `McpError::NoServersConfigured`.
  - Per D-06: Iterate servers sequentially (1-3 servers, sub-second total).
  - For each URL, call `connect_and_discover()`. Per MCP-06: If ANY server fails, return the error immediately (fail fast).
  - Per D-08: Prefix all tool names with host-based identifier: `{host_prefix}__{tool_name}`.
  - Per D-07: Connections are ephemeral. Only the ToolsWithRouting result is returned (serializable).
  - Build the routing HashMap mapping each prefixed tool name to its server URL.

b) `async fn connect_and_discover(server_url: &str) -> Result<Vec<pmcp::types::ToolInfo>, McpError>`:
  - Per MCP-01: Create `StreamableHttpTransport` with `StreamableHttpTransportConfig`:
    ```rust
    let url = url::Url::parse(server_url)
        .map_err(|_| McpError::InvalidUrl(server_url.to_string()))?;
    let config = StreamableHttpTransportConfig {
        url,
        extra_headers: vec![],
        auth_provider: None,  // Per D-05: no auth for PoC
        session_id: None,
        enable_json_response: false,
        on_resumption_token: None,
        http_middleware_chain: None,
    };
    let transport = StreamableHttpTransport::new(config);
    ```
  - Create client with `Client::with_info(transport, Implementation::new("durable-mcp-agent", "0.1.0"))`.
  - Call `client.initialize(ClientCapabilities::default()).await` -- map errors to `McpError::InitializationFailed`.
  - Per MCP-02: Call `client.list_tools(None).await`, handle pagination via `next_cursor` loop. Map errors to `McpError::DiscoveryFailed`.
  - Return the collected `Vec<ToolInfo>`.

c) `fn translate_mcp_tool(tool_info: &pmcp::types::ToolInfo, prefix: &str) -> UnifiedTool`:
  - Per MCP-03 and D-08: Create `UnifiedTool` with:
    - `name`: `format!("{prefix}__{}", tool_info.name)`
    - `description`: `tool_info.description.clone().unwrap_or_default()`
    - `input_schema`: `clean_tool_schema(&tool_info.input_schema)` (reuse from `crate::llm::transformers::utils`)

d) `fn extract_host_prefix(server_url: &str) -> Result<String, McpError>`:
  - Parse URL with `url::Url::parse(server_url)`.
  - Extract host with `.host_str()`, return `McpError::InvalidUrl` if None.
  - Take the first segment before the first dot: `host.split('.').next().unwrap_or(host)`.
  - Return the prefix string.
  - Example: `"https://calc-server.us-east-1.amazonaws.com/mcp"` -> `"calc-server"`

e) `pub fn resolve_tool_call(prefixed_name: &str, routing: &HashMap<String, String>) -> Result<(String, String), McpError>`:
  - Per D-09: Look up prefixed_name in routing map. Return `McpError::UnknownTool` if not found.
  - Strip prefix using `splitn(2, "__")` (not `split("__")`) to handle tool names that contain `__`. Per Pitfall 4 from RESEARCH.md.
  - Return `(server_url, original_tool_name)`.

Add comprehensive tests in `#[cfg(test)] mod tests`:
