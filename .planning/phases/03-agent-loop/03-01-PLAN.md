**Public function `agent_handler`:**
```
pub async fn agent_handler(
    event: AgentRequest,
    ctx: DurableContextHandle,
    llm_service: UnifiedLLMService,
) -> DurableResult<AgentResponse>
```

Implementation steps (each numbered per LOOP/MCP requirements):

1. **Load config (CONF-04, existing pattern):**
   - Read `AGENT_REGISTRY_TABLE` env var, default to `"AgentRegistry"`
   - Clone `event.agent_name` and `event.version` into owned strings for the closure
   - `ctx.step(Some("load-config"), move |_| async move { load_agent_config(&table_name, &agent_name, &version).await.map_err(|e| -> Box<dyn std::error::Error + Send + Sync> { Box::new(e) }) }, None).await?`
   - Result type: `AgentConfig`

2. **Map provider config (per D-01):**
   - The `load_agent_config` already returns a fully populated `AgentConfig` with `provider_config`. No extra step needed.

3. **Discover tools (MCP-02, existing pattern):**
   - Clone `config.mcp_server_urls`
   - Wrap `discover_all_tools` in `ctx.step(Some("discover-tools"), ...)` with no retry (tool discovery should fail fast per MCP-06)
   - Handle the case where `config.mcp_server_urls` is empty: skip tool discovery, use empty `ToolsWithRouting { tools: vec![], routing: HashMap::new() }`. This supports agents that don't use MCP tools (pure LLM conversation).
   - Result type: `ToolsWithRouting`

4. **Establish MCP connections (D-03, D-05):**
   - OUTSIDE durable steps: `establish_mcp_connections(&config.mcp_server_urls).await?`
   - Only call if `mcp_server_urls` is non-empty, otherwise use empty `Arc::new(HashMap::new())`
   - Convert McpError to DurableError::Internal for the handler return type

5. **Agent loop (LOOP-01, LOOP-04, LOOP-05, LOOP-06):**
   ```rust
   let mut messages: Vec<UnifiedMessage> = event.messages;
   let max_iterations = config.parameters.max_iterations;

   for i in 0..max_iterations {
       let iteration_result = ctx.run_in_child_context(
           Some(&format!("iteration-{i}")),
           |child_ctx| {
               let llm = llm_service.clone();
               let cfg = config.clone();
               let tools = tools_with_routing.clone();
               let msgs = messages.clone();
               let clients = mcp_clients.clone();
               async move {
                   execute_iteration(child_ctx, &llm, &cfg, &tools, &msgs, &clients, i).await
               }
           },
           None,
       ).await?;

       // Append assistant message to history (LOOP-05)
       messages.push(iteration_result.assistant_message.clone());

       // Check if done (LOOP-07)
       if iteration_result.is_final {
           return Ok(AgentResponse {
               response: iteration_result.llm_response,
           });
       }

       // Append tool results to history (LOOP-05)
       if let Some(tool_results_msg) = &iteration_result.tool_results_message {
           messages.push(tool_results_msg.clone());
       }
   }

   // Max iterations exceeded (LOOP-06)
   Err(DurableError::Internal(format!(
       "Agent exceeded max iterations ({max_iterations}) without completing"
   )))
   ```

**Private function `execute_iteration`** (LOOP-02, LOOP-03, MCP-04, MCP-05):
```
async fn execute_iteration(
    ctx: DurableContextHandle,
    llm: &UnifiedLLMService,
    config: &AgentConfig,
    tools: &ToolsWithRouting,
    messages: &[UnifiedMessage],
    mcp_clients: &McpClientCache,
    iteration: u32,
) -> DurableResult<IterationResult>
```

Steps inside:
1. Build LLMInvocation via `build_llm_invocation(config, messages, &tools.tools)`
2. Clone llm and invocation for the step closure
3. Create ExponentialBackoff: `max_attempts(3)`, `initial_delay(Duration::seconds(2))`, `max_delay(Duration::seconds(30))`, `backoff_rate(2.0)` (LOOP-02)
4. Create `StepConfig::<LLMResponse>::new().with_retry_strategy(Arc::new(retry))`
5. `ctx.step(Some("llm-call"), move |_| async move { llm_clone.process(invocation).await.map_err(|e| -> Box<dyn std::error::Error + Send + Sync> { Box::new(e) }) }, Some(step_config)).await?`
6. Build assistant message via `llm_response_to_assistant_message(&llm_response)`
7. Check `is_final`: `llm_response.metadata.stop_reason.as_deref() == Some("end_turn")`
8. If is_final OR function_calls is None/empty, return `IterationResult { is_final: true, tool_results_message: None, ... }`
9. Execute tools via `ctx.map()` (LOOP-03, MCP-04, MCP-05):
   - Get function_calls from llm_response
   - Clone routing and clients for the map closure
   - `ctx.map(Some("tools"), function_calls, move |call, _item_ctx, _idx| { let r = routing.clone(); let c = clients.clone(); async move { execute_tool_call(&call, &r, &c).await.map_err(|e| DurableError::Internal(e.to_string())) } }, None).await?`
   - Extract results via `batch_result.values()`
10. Build tool results message via `build_tool_results_message(tool_results)`
11. Return `IterationResult { is_final: false, tool_results_message: Some(msg), ... }`
