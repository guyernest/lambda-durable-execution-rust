**1. Doc header (//! comments):**
```
//! Example 65: Durable MCP Agent
//!
//! Demonstrates building an MCP agent (LLM + tool loop) using AWS Lambda
//! Durable Execution for checkpointed, replay-safe workflows.
//!
//! The agent:
//! 1. Connects to configured MCP servers and discovers tools
//! 2. Calls the Anthropic Messages API with the tool schemas
//! 3. Executes any tool_use blocks via MCP call_tool
//! 4. Repeats until the LLM returns end_turn
//!
//! Each LLM call is a durable `ctx.step()` with retry. Tool executions
//! use `ctx.map()` for parallel, replay-safe execution. Each iteration
//! is wrapped in `ctx.run_in_child_context()` for deterministic replay.
//!
//! For task-aware MCP servers (long-running tools), the agent uses
//! `ctx.wait_for_condition()` to poll task status with zero compute cost
//! during waits -- Lambda suspends between poll attempts.
//!
//! ## Prerequisites
//!
//! - AWS Lambda with Durable Execution enabled
//! - `ANTHROPIC_API_KEY` environment variable set
//! - One or more MCP servers accessible via HTTP
//!
//! ## Deployment
//!
//! This example runs as an AWS Lambda function. Deploy with SAM:
//! ```bash
//! sam build && sam deploy --guided
//! ```
//!
//! ## Invocation
//!
//! ```json
//! {
//!   "prompt": "What tools do you have? Try using one.",
//!   "mcp_server_urls": ["https://your-mcp-server.example.com/mcp"],
//!   "model": "claude-sonnet-4-20250514",
//!   "max_iterations": 10
//! }
//! ```
```

**5. agent_handler() function -- the core handler:**
Simplified from `handler.rs`. Signature: `async fn agent_handler(event: AgentInput, ctx: DurableContextHandle) -> DurableResult<AgentOutput>`

Steps:
a. Read config from event (inline -- no DynamoDB, per Claude's discretion on config approach)
b. Read API key from `ANTHROPIC_API_KEY` env var
c. Discover tools from MCP servers via `ctx.step("discover-tools", ...)` -- call `discover_tools()` helper
d. Establish MCP connections OUTSIDE durable steps (per Pitfall 1) -- call `connect_mcp_clients()` helper
e. Agent loop with `ctx.run_in_child_context("iteration-{i}", ...)` per iteration
f. Inside each iteration:
   - Build messages array (system prompt + conversation history)
   - Call LLM via `ctx.step("llm-call", ...)` with ExponentialBackoff retry (max_attempts: 3, initial_delay: 2s, max_delay: 30s, backoff_rate: 2.0)
   - Parse response: check stop_reason == "end_turn" or no tool_use blocks -> return final
   - Extract tool_use blocks -> execute via `ctx.map("tools", tool_calls, ...)` calling `execute_tool_call()` helper
   - Build tool_result messages and append to history
g. Accumulate token counts and tool names across iterations (per SDK-03)
h. Structured logging with tracing::info! per iteration (per SDK-03):
   ```rust
   info!(
       iteration = i,
       input_tokens = response.usage.input_tokens,
       output_tokens = response.usage.output_tokens,
       total_input_tokens,
       total_output_tokens,
       tool_count = tool_calls.len(),
       "Iteration complete"
   );
   ```
i. On max_iterations exceeded: return DurableError::Internal

**6. Helper functions:**

`call_anthropic()` -- Direct reqwest call to Anthropic Messages API:
- Takes: `&reqwest::Client`, `api_key: &str`, `model: &str`, `messages: &[serde_json::Value]`, `tools: &[serde_json::Value]`
- POST to `https://api.anthropic.com/v1/messages`
- Headers: `x-api-key`, `anthropic-version: 2023-06-01`, `content-type: application/json`
- Body: `{ model, max_tokens: 4096, messages, tools }`
- Returns: `Result<AnthropicResponse, Box<dyn std::error::Error + Send + Sync>>`

`discover_tools()` -- Discover and translate tools from MCP servers:
- Takes: `&[String]` (server URLs)
- For each URL: create temp client, call `list_tools()`, translate to Anthropic tool format
- Returns tool schemas as `Vec<serde_json::Value>` (Anthropic format: name, description, input_schema)
- Also returns routing map: `HashMap<String, String>` (prefixed_name -> server_url)
- Use host prefix extraction: `url.host_str().split('.').next()` for prefix
- Prefixed name format: `{prefix}__{tool_name}` (per existing pattern)
- Return type: a `ToolsWithRouting` struct defined inline

`connect_mcp_clients()` -- Establish live MCP connections:
- Takes: `&[String]` (server URLs)
- For each URL: parse, create StreamableHttpTransport, create Client with Implementation::new("durable-mcp-agent-example", "0.1.0"), initialize with `ClientCapabilities::full()` (includes tasks support, per D-08)
- Returns: `Arc<HashMap<String, Client<StreamableHttpTransport>>>`
- CRITICAL: This function runs OUTSIDE durable steps (connections are ephemeral per Lambda invocation)

`execute_tool_call()` -- Execute a single tool call via MCP:
- Takes: prefixed tool name, arguments (serde_json::Value), routing map ref, client cache ref
- Resolve: use `split_once("__")` to get (prefix, original_name), look up server_url from routing
- Call: `client.call_tool(original_name, arguments)` (NOT call_tool_with_task -- that is Plan 02)
- Extract text from Content::Text variants, join with newline
- Return: `ToolCallResult { tool_use_id, content, is_error }` (inline struct)
- MCP errors (is_error: true) passed through as successful results (LLM decides recovery)

`extract_tool_calls()` -- Parse tool_use blocks from AnthropicResponse:
- Filter content blocks for ToolUse variant
- Return Vec of (id, name, input) tuples or a small struct
