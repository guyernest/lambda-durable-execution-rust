**execute-agent/handler.ts** -- Full implementation following execute-test-scenario pattern (per D-08):
- Import `AppSyncResolverHandler`, `LambdaClient`, `InvokeCommand`, `DynamoDBClient`, `DynamoDBDocumentClient`, `GetCommand`, `PutCommand`, `UpdateCommand`, `QueryCommand`, `uuid`
- Read env vars: `AGENT_LAMBDA_FUNCTION_NAME`, `AGENT_CONFIGS_TABLE`, `AGENT_EXECUTIONS_TABLE`, `LLM_MODELS_TABLE`, `MCP_SERVERS_TABLE`
- `executeAgent` function:
  1. Look up AgentConfig by ID from AGENT_CONFIGS_TABLE
  2. Look up LLMModel by config's modelId from LLM_MODELS_TABLE
  3. Look up MCP server endpoints from MCP_SERVERS_TABLE for each mcpServerIds entry
  4. Generate execution ID with uuid v4
  5. Create AgentExecution record with status='pending', startedAt=now
  6. Build agent invocation payload: `{ agent_config_id, execution_id, agent_name, instructions, model (provider + modelId), temperature, max_tokens, max_iterations, mcp_servers (array of {name, endpoint}), input, executions_table (so agent can write updates) }`
  7. Invoke Lambda with `InvocationType: 'Event'` (async), `Payload: JSON.stringify(payload)`
  8. Update execution status to 'running'
  9. Return `{ executionId, status: 'running' }`
- Handler switches on fieldName `executeAgent`
- Use the identity pattern: `const identity = event.identity as any; const userId = identity?.sub || identity?.claims?.sub || 'unknown';`

The handler implements the `getAgentExecutionDetail` field:
1. Look up AgentExecution by ID from AGENT_EXECUTIONS_TABLE
2. If execution not found, throw Error
3. If execution has `durableExecutionId`:
   - Call AWS Durable Execution API via `GetDurableExecution` to get checkpoint data
   - NOTE: For the initial implementation, the conversation history is reconstructed from the AgentExecution record fields. Full durable execution checkpoint inspection is a stretch goal.
   - For now, return the execution record data: status, iterations, token counts, tools called, duration, error
4. Build a `conversationHistory` array from the execution data:
   - Include the input as a user message
   - If output exists, include as assistant message
   - toolsCalled as metadata
5. Return the result matching the customType schema:
```typescript
{
  id: execution.id,
  status: execution.status,
  conversationHistory: [
    { role: 'user', content: execution.input },
    ...(execution.output ? [{ role: 'assistant', content: execution.output }] : []),
  ],
  checkpoints: null, // Future: populated from Durable Execution API
  iterations: execution.iterations || 0,
  inputTokens: execution.inputTokens || 0,
  outputTokens: execution.outputTokens || 0,
  toolsCalled: execution.toolsCalled || [],
  durationMs: execution.durationMs || 0,
  errorMessage: execution.errorMessage || null,
}
```
